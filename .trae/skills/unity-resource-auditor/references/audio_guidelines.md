# 音频优化指南 (Audio Optimization Guidelines)

## 核心原则
音频主要影响 CPU 负载和内存占用。合理的导入设置可以显著降低这些开销。

## 检查清单 (Checklist)

### 1. Load Type (加载方式)
*   **规则**:
    *   **Decompress On Load**: 适用于小文件（< 0.2 MB），如 UI 音效、短促的攻击声。会增加内存占用，但无 CPU 解码开销。
    *   **Compressed In Memory**: 适用于中等文件（0.2 - 0.5 MB），如环境音效。会增加 CPU 开销，但节省内存。
    *   **Streaming**: 适用于大文件（> 0.5 MB），如背景音乐、长语音。几乎不占内存，但会有持续的 CPU/IO 开销。

### 2. Compression Format (压缩格式)
*   **规则**:
    *   **Vorbis**: 默认推荐，质量较好，适合大多数情况。
    *   **ADPCM**: 适用于音质要求不高的环境音效，解码极快。
    *   **PCM**: 无损格式，体积最大，仅用于极短且频繁播放的音效（如 UI 点击）。

### 3. Force To Mono (强制单声道)
*   **规则**:
    *   **3D 音效**: **必须开启**。3D 音效在空间中定位时会自动混音为立体声，源文件保留立体声不仅浪费内存，还会增加 CPU 计算量。
    *   **2D 背景音乐**: 除非确实需要立体声效果，否则也建议开启以节省空间。

### 4. Sample Rate Setting (采样率设置)
*   **规则**:
    *   **Optimize Sample Rate**: 默认开启。Unity 会自动根据压缩格式调整采样率。
    *   **Override Sample Rate**: 手动设置（如 22050 Hz 或 44100 Hz）。对于移动端，通常 22050 Hz 已足够清晰，且文件体积减半。

## 常见问题
*   **Preload Audio Data**:
    *   **规则**: 默认开启。确保音频数据在场景加载时预先加载进内存，避免播放时的卡顿。
    *   **例外**: 对于非常大的流式音频（Streaming），可以关闭以加快场景加载速度。
*   **Quality**:
    *   **规则**: 压缩质量通常设为 70% 左右即可满足需求，过高会增加文件体积。
